# Avareno Connect Security Rules

Avareno Connect and Custom Connector Builder are high-risk. They can expose third-party accounts, mailboxes, files, devices, households, and sensitive documents.

No connector may ship to production without privacy review, security review, and documentation.

## Authentication Methods

Allowed or preferred:

- OAuth with narrow scopes
- API tokens generated by the provider
- app passwords where the provider supports scoped app passwords
- webhooks
- import addresses
- user-selected file uploads

Not allowed:

- collecting normal third-party usernames/passwords
- storing plaintext credentials
- asking users to paste session cookies
- scraping authenticated websites with user credentials in MVP
- arbitrary user-provided code execution in MVP

## Permissions

- Read-only by default.
- Show scopes clearly before connection.
- Explain what will be imported.
- Prefer preview before import.
- Do not import entire mailboxes, drives, archives, or unrelated files by default.
- Prefer user-confirmed import batches.
- Users must be able to disconnect.
- Users must be able to delete synced data.

## Token Storage

- Tokens/secrets must be encrypted at rest before production.
- Tokens/secrets must be stored server-side only.
- Tokens/secrets must never be exposed to frontend code.
- Tokens/secrets must never be logged.
- Tokens/secrets must never be included in AI prompts.
- Token deletion must happen on disconnect/account deletion.

## Custom Connector URL Protection

Custom connector URLs must be protected against SSRF.

Block:

- localhost and loopback ranges
- private IPv4 ranges
- private IPv6 ranges
- link-local addresses
- multicast/broadcast ranges
- internal hostnames
- cloud metadata endpoints such as `169.254.169.254`
- redirects into blocked ranges

Require:

- DNS resolution checks before request
- redirect validation on every hop
- HTTPS where practical
- request timeouts
- response size limits
- content-type allowlist where practical
- rate limits
- safe error messages

## Sync Behavior

- Do not run broad background sync by default.
- Use explicit user action for first import.
- Provide sync preview where practical.
- Record safe sync logs:
  - provider
  - time
  - status
  - item count
  - safe error code
- Do not log raw payloads, tokens, email bodies, full documents, addresses, order numbers, or private files.

## Connector Disconnect

Disconnect must:

- revoke provider token if supported
- delete local token/secret
- stop background jobs
- show last sync status
- offer deletion of synced data where appropriate
- keep only minimal audit metadata if legally/product necessary

## Rate Limits And Abuse

- Rate-limit connector calls by user/provider.
- Add provider timeouts.
- Add retry limits with backoff.
- Avoid scanning networks or providers without explicit user action.
- Local network discovery must be opt-in and limited.

## MVP Limits

For MVP:

- no arbitrary user code
- no full mailbox import
- no full filesystem/cloud drive import
- no write actions to third-party providers unless explicitly reviewed
- no connector import into Private Vault without explicit user action

## Current Engineering Foundation

- `backend/app/security/connectors.py` contains URL/redirect validation helpers for future custom connector work.
- `require_encrypted_connector_secret_storage()` intentionally raises until encrypted server-side token storage is implemented.
- These helpers are not a finished connector platform and must not be presented as production-ready Connect.
